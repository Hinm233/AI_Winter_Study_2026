# 第5天：Python高效语法+Git冲突解决+Linux包安装/换源
## 核心知识点
1. Python：**列表推导式/字典推导式**（AI数据处理提速必备，替代繁琐for循环）、**匿名函数lambda**+**高阶函数map/filter/zip**（数据处理高频组合）；
2. Git：**分支冲突解决**（团队开发必遇场景，核心是手动修改冲突文件，重新提交）；
3. Linux：**pip/conda换国内源**（解决包安装慢的问题，必做配置）、`pip list/pip show`（查看已装包/包信息）、`conda clean -y`（清理conda缓存）。

## 实操要求（按顺序做，无遗漏）
### 一、Python核心（1.5h）
1. **推导式核心（列表+字典）**：
   - 列表推导式：生成1-20的偶数列表、生成成绩列表中≥90的成绩列表，替代传统for循环（一行代码实现）；
   - 字典推导式：将两个列表`names = ["张三","李四","王五"]`、`scores = [90,85,95]`合并为字典`{姓名:成绩}`，一行代码实现；
   - 带条件的推导式：生成1-100的奇数的平方列表，`[x**2 for x in range(1,101) if x%2==1]`。
2. **lambda+map/filter/zip**：
   - lambda：定义匿名函数，计算两个数的和、计算成绩的平方，调用并打印结果；
   - map：用map+lambda将成绩列表`[85,92,78]`转换为百分制+5的列表，用`list()`转换为可迭代对象；
   - filter：用filter+lambda筛选成绩列表中≥80的成绩，用`list()`转换并打印；
   - zip：用zip将姓名和成绩列表打包，遍历并打印每一组（姓名,成绩）。
3. 综合练习：用推导式+lambda+map，实现“从成绩字典中筛选出Python成绩≥90的学生，生成新字典”。

### 二、Git核心（0.4h）
1. **模拟分支冲突（必做，掌握解决方法）**：
   - 切到dev分支：`git checkout dev`，修改`stu_score.txt`文件，添加一行「赵六:92」，提交：`git add .`→`git commit -m "dev：添加赵六成绩"`；
   - 切回master分支：`git checkout master`，**修改同一个文件的同一行**，比如添加一行「赵六:90」，提交：`git add .`→`git commit -m "master：添加赵六成绩"`；
   - 执行`git merge dev`，触发**分支冲突**（终端显示`CONFLICT (content): Merge conflict in stu_score.txt`）；
2. **解决冲突**：
   - 打开冲突文件`stu_score.txt`，看到冲突标记`<<<<<<< HEAD`（master分支内容）、`=======`（分隔符）、`>>>>>>> dev`（dev分支内容）；
   - 手动修改文件，保留需要的内容，**删除所有冲突标记**，保存文件；
   - 执行`git add stu_score.txt`→`git commit -m "解决分支冲突：统一赵六成绩为92"`→`git push`，完成冲突解决。

### 三、Linux核心（0.1h）
1. **永久换源（必做，一次配置终身受用）**：
   - Conda换源：在Git Bash中执行命令，添加清华源（复制粘贴即可，无拼写错误）；
   - Pip换源：在用户目录下创建`pip`文件夹，新建`pip.ini`文件，添加阿里/清华源，保存；
2. 激活ai_base环境：`conda activate ai_base`，执行`pip install opencv-python`，测试换源后安装速度（明显变快）；
3. 执行`pip list`，查看已装包；执行`pip show numpy`，查看numpy的版本、路径等信息；
4. 执行`conda clean -y`，清理conda缓存，释放磁盘空间。

## 成果校验
1. Python：能熟练使用推导式替代for循环，能独立组合lambda+map/filter/zip处理数据，一行代码实现复杂数据转换，无报错；
2. Git：能主动触发分支冲突，手动修改冲突文件，完成冲突解决，合并后代码无错误，远程仓库同步成功；
